The Problem

eSIM plans are non-transferable. Buyers who can't travel lose their money. Sellers can't resell unused plans.

The Solution

NFT vouchers that encode real eSIM entitlements. Trade freely before activation. Activate only after purchase.

Smart Contract

contracts/ESIMNFT.sol — ERC-721 + Enumerable (OpenZeppelin v5, Solidity 0.8.28)

Token Standard
ERC-721 + Enumerable
Fully compatible with OpenSea, Blur, and all NFT marketplaces.
Plan Metadata
On-Chain
Provider, planId, country, data (bytes), validity (days) — stored per token.
Activation
Soulbound Lock
Activated tokens cannot be transferred. ERC-5192 style gating via _update.
Operator Role
Gasless UX
Platform activates on behalf of users. No ETH needed from buyer.
EIP-712 Permits
Off-Chain Signatures
User signs an activation permit; platform submits it.
Burning
Cleanup
Activated tokens can be burned by their owner.

Lifecycle

From eSIM provider purchase to mobile activation.

1. Mint → 2. List → 3. Buy → 4. Activate → 5. Use
StepWhoActionOn-Chain
1. MintPlatformBuys eSIM plan from provider (e.g. Airalo), calls mint()✅ Token created
2. ListPlatformLists NFT on OpenSea at cost + margin—
3. BuyUserPurchases NFT on OpenSea (standard ERC-721 transfer)✅ Ownership changes
4. ActivateUserRequests activation on platform → provider API called → activate()✅ Locked (non-transferable)
5. UseUserInstalls eSIM profile on phone using activation code—
Key Property
Transferable ⇄ Non-Transferable
Before activation: free trade on any marketplace. After activation: permanently soulbound.

Metadata API

Vercel serverless function at api/metadata/[tokenId].ts

Reads On-Chain

Uses viem to call getPlan(tokenId) live on Abstract Testnet. Returns standard ERC-721 metadata that OpenSea renders.
